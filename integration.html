Common questions

HR software integration FAQ

EKTA HR's Tally integration uses Tally's XML/REST interface to push processed payroll data — including salary components, PF, ESI, and professional tax — directly into the correct Tally ledgers. Setup takes under 2 hours: you configure your ledger mapping once, and every subsequent payroll run syncs automatically. No manual exports, no CSV imports.

EKTA HR natively supports ZKTeco, eSSL, and Matrix biometric devices — the three most widely deployed brands in Indian workplaces. Punch data from fingerprint, face recognition, and card-based devices syncs directly into EKTA HR's attendance module in real time. For other device brands, our PUSH API allows custom integration.

HRMS API integration means connecting your HRMS to other business systems (ERP, accounting software, custom apps) via programmatic APIs rather than manual data export. For EKTA HR's pre-built integrations with Tally, SAP, QuickBooks, Okta, and others, no developer is needed — they're configured through the admin panel. For custom integrations using our REST API, your IT team or a system integrator can connect any system using our API documentation.

In EKTA HR, attendance data from biometric devices, the mobile app, or manual marking is processed against your shift schedules to calculate present days, absences, LOP (Loss of Pay), overtime, and late marks. This processed attendance data feeds directly into the payroll engine — no manual transfer needed. Payroll picks up the attendance figures for the pay period automatically when you run payroll, applying your configured LOP rules, overtime rates, and regularisation policies.

EKTA HR can maintain bidirectional sync with multiple ERP or HCM systems simultaneously — useful for organisations mid-migration between SAP and Oracle, or companies running SAP for finance and Oracle HCM for global workforce management. Data mappings are configured independently for each integration.

Absolutely. Google Workspace SSO is available on all EKTA HR plans and takes about 30 minutes to configure — no dedicated IT team required. Okta SSO with SCIM provisioning is available for larger teams that need automated user lifecycle management. Both options eliminate the need for employees to manage a separate HRMS password.

Key things to verify before go-live: (1) data field mapping between both systems is complete and reviewed, (2) sync direction is configured correctly — one-way or bidirectional per data type, (3) conflict resolution rules are set for when data changes in both systems simultaneously, (4) historical data migration scope is agreed upon, (5) a rollback plan exists for the first 30 days, (6) user acceptance testing is done with real payroll data from a previous month, (7) IT and HR teams are trained on how to handle sync errors.
